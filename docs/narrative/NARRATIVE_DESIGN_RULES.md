# RED FRONTIER — Narrative Design Rules
**Version 1.0 | Act I Reference**

> หลักการสำคัญ: ตัวละครตอบสนองต่อเหตุการณ์ตาม "สิ่งที่รู้ ณ ตอนนั้น" ไม่ใช่ตาม "สิ่งที่ผู้เล่นอ่าน wiki มาแล้ว"

## 12 กฎหลักการสร้างบทสนทนาและตัวเลือก

### Rule 1 — Knowledge-First
ตัวละครห้ามตอบสนองเกินกว่าสิ่งที่รู้จริง  
ถ้ายังไม่รู้ว่าเกิดอะไรขึ้น → บทพูดและตัวเลือกต้องสะท้อน "ความสับสน / ความลังเล / การตีความจากข้อมูลจำกัด"

### Rule 2 — Human Reaction First
ก่อนตัดสินใจเชิงกลยุทธ์ ให้คำนึงถึงปฏิกิริยาแบบมนุษย์ก่อน:
- งง / ชะงัก / ตั้งสติ
- ปฏิเสธความจริงชั่วคราว
- สัญชาตญาณเอาตัวรอด
- ทำตามความเคยชินของอาชีพ

### Rule 3 — Class Response
ตัวละครต่างคลาสตอบสนองต่างกันแม้อยู่ในเหตุการณ์เดียวกัน  
(ดูตาราง Class Response Matrix ด้านล่าง)

### Rule 4 — Stress Response
ในเหตุการณ์ฉุกเฉิน ตัวเลือกแรกๆ ต้องสะท้อนการตอบสนองภายใต้ความกดดัน:
- หาที่กำบัง / ตั้งสติ / ตรวจ HUD
- มองหาทางหนี / ติดต่อใครบางคน
- ปิดประตู / ล็อกทาง
> ห้ามเปิดด้วย aggressive choice เว้นแต่คลาส/บริบทเหมาะจริง

### Rule 5 — Action Plausibility
ตัวเลือกต้องสมจริงต่อ "คนแบบนั้น" ใน "ช่วงเวลานั้น"
- หมอที่ยังไม่รู้สถานการณ์ → ไม่ควรมี choice แรกเป็น "เตรียมอาวุธสู้"
- นักวิทยาศาสตร์ → ไม่ควรเริ่มด้วย "บุกยิง" ถ้ายังมีเวลาวิเคราะห์
- คนทั่วไป → อาจ freeze หรือไม่ทำอะไรถูกในช่วงแรก

### Rule 6 — Escalation
ตัวเลือกควรไล่ระดับจาก:
```
รับรู้ → ตั้งสติ → ประเมิน → ตัดสินใจ → ลงมือ
```
ห้ามกระโดดจาก "เพิ่งรู้ว่ามีภัย" ไป "สู้ตาย" ทันที

### Rule 7 — Tone Consistency
โทนต้องสอดคล้องกับ **Hard Sci-Fi Survival RPG**  
ความกลัว / ความไม่แน่ใจ / ความเหนื่อย / ความกดดันจาก Mars ต้องรู้สึกได้ในบท  
ห้ามให้ตัวละครพูดหรือเลือกแบบ heroic fantasy

### Rule 8 — Contextual Choice Framing
ข้อความตัวเลือกต้องใช้ภาษาที่เข้ากับ "ระดับความเข้าใจของตัวละคร"
- ❌ `สู้กับ Strike Team` (omniscient)
- ✅ `เตรียมป้องกันตัวถ้าพวกเขาบุกเข้ามา` (perception filter)
- ❌ `เตรียมอาวุธรับมือ Strike Team` (รู้ชื่อหน่วยก่อนจะเห็น)
- ✅ `ตรวจ HUD และหาทางออก` (ปฏิกิริยาแรก)

### Rule 9 — No Omniscient Choices
ห้ามสร้างตัวเลือกที่เหมือนรู้ผลลัพธ์ล่วงหน้า  
เขียนจากมุมมองตัวละคร ไม่ใช่มุมมองคนเขียนระบบ

### Rule 10 — Follow-up Logic
ถ้าตัวละครเริ่มจากความงง → ตัวเลือก follow-up ต้องสะท้อนการเปลี่ยนผ่านตามธรรมชาติ
- ตรวจ HUD → รู้ว่าถูกตามล่า → เริ่มหนี / ซ่อน / ส่งข้อมูล
- ได้ยินเสียงฝีเท้า → ระวัง → คิดว่าจะสู้หรือหนี

### Rule 11 — Experience Override
พื้นหลังเฉพาะสามารถ override reaction ปกติได้
- Medic ที่เคยเป็น field medic → tactical กว่า Medic ทั่วไป
- Scientist ที่เคยอยู่ Hephaestus-7 → ไม่งงเท่าคนอื่น
- Engineer ที่เคยผ่าน sabotage → ระแวงเร็ว

### Rule 12 — Early-Scene Choice Design
สำหรับฉากแรกๆ หรือผู้เล่นยังไม่เข้าใจสถานการณ์:
- ต้องมี choice ที่ช่วยให้ "เข้าใจโลก" ไม่ใช่แค่ "เลือกผลลัพธ์"
- ต้องมี choice สะท้อนความสับสน / การสังเกต / การตั้งสติ
- choice เชิงกลยุทธ์หนักควรมาทีหลัง หรือเป็น class-specific

## Class Response Matrix

| สถานการณ์ | Ranger | Medic | Scientist | Engineer | Augmented |
|-----------|--------|-------|-----------|----------|-----------|
| **เพิ่งรู้ว่ามีภัย** | ประเมิน threat, หาจุดตั้งรับ | ตั้งสติ, ตรวจ HUD | หาต้นตอ Alert, ตรวจข้อมูล | ล็อกประตู, ถ่วงเวลา | ตรวจ anomaly, network scan |
| **ถูกไล่ล่า** | ถอย tactical, หาจุดซุ่ม | หาที่กำบัง, หนี | ซ่อน/ส่งข้อมูลก่อน | ปิดประตู, หาท่อระบาย | spoof trace, หนี |
| **เจอศัตรูหน้าตัว** | ประเมิน intent, เตรียมตั้งรับ | หาทางเจรจา, freeze ชั่วขณะ | พยายามเจรจา, หาทางหนี | หาทางหนีระบบ, เจรจา | ตรวจ comms ศัตรู, หาทางออก |
| **ข้อมูลสำคัญอยู่ในมือ** | ซ่อน/ส่งก่อนหนี | รักษาไว้, หาคนที่ไว้วางใจได้ | ส่งออก / วิเคราะห์เพิ่ม | ซ่อนในระบบ, encrypt | ซ่อนใน neural buffer, encrypt |

## Escalation Template

```
ฉาก [ชื่อ] — ตัวละครรู้แค่: [X]
ปฏิกิริยาธรรมชาติคลาส [Y]: [Z]

Choice 1: [รับรู้ / ตั้งสติ]          ← ทุก class เข้าถึงได้
Choice 2: [ประเมิน / หาข้อมูล]        ← เหมาะ class [A, B]
Choice 3: [ตัดสินใจ / ลงมือ — อ่อน]   ← ทั่วไป
Choice 4: [ลงมือ — หนัก / aggressive] ← class-specific หรือ last resort
```

## Choice Writing Checklist

ก่อน finalize choice ทุก choice ให้ตรวจ:

- [ ] ตัวละครรู้เรื่องนี้จริงๆ ณ ตอนนั้นหรือเปล่า?
- [ ] ภาษาของ choice สะท้อน perception ของตัวละคร ไม่ใช่ meta-knowledge?
- [ ] choice นี้สมเหตุสมผลกับ ALL classes หรือควรเป็น class-specific?
- [ ] ลำดับ choice ไล่ระดับจาก safe → risky ตามธรรมชาติ?
- [ ] โทนสอดคล้องกับ Hard Sci-Fi Survival RPG?
- [ ] follow-up choices สะท้อนการเปลี่ยนผ่านที่สมเหตุสมผล?

## ตัวอย่าง: NODE_DATA_FOUND (ก่อน/หลัง)

### ❌ ก่อน (ปัญหา)
```
Choice 1: เตรียมอาวุธและรับมือกับ Strike Team  ← omniscient, aggressive เกิน
Choice 2: หลบหนีผ่าน Maintenance Tunnel        ← ดี แต่ลำดับไม่ถูก
Choice 3: ส่งข้อมูลไปยัง ISU HQ               ← ดี
```

**ปัญหา:** ผู้เล่นเพิ่งเห็น Alert บน HUD แต่ choice แรกคือ "เตรียมสู้ Strike Team"  
— ตัวละครยังไม่รู้ว่า Strike Team มีกี่คน จะมาจากไหน มี intent อะไร  
— ไม่มีช่วง "ตั้งสติ / ประเมิน" เลย

### ✅ หลัง (ตาม Rules)
```
Choice 1: ตรวจ HUD และหาทางออกฉุกเฉิน        ← Escalation step 1, ทุก class
Choice 2: หลบหนีผ่าน Maintenance Tunnel       ← Escape first
Choice 3: ส่งหรือซ่อนข้อมูลก่อนที่จะสายเกิน ← Scientist/Eng/Aug logic
Choice 4: เตรียมป้องกันตัว ถ้าไม่มีทางหนีแล้ว ← สุดท้าย, class-leaning
```

## ตัวอย่าง: NODE_PROMETHEUS_ALERT (ก่อน/หลัง)

### ❌ ก่อน
```
Choice 1: สู้! (นำไปสู่ Combat)              ← ภาษาเกม, omniscient
Choice 2: เจรจาและแกล้งทำเป็นว่าเข้าใจผิด  ← ดี
Choice 3: วิ่งหนีโดยไม่ลังเล               ← ดี แต่ลำดับผิด
```

### ✅ หลัง
```
Choice 1: ถอยหลังช้าๆ ขณะประเมินสถานการณ์  ← สัญชาตญาณ, ทุก class
Choice 2: ยกมือและพยายามพูดคุย              ← Human Reaction
Choice 3: วิ่งหนีทันที                      ← Escape
Choice 4: ตั้งรับและต่อสู้                  ← สุดท้าย, ถ้าถูกบีบ
```
